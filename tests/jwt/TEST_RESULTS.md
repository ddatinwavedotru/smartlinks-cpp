## Детали по тест-сьютам

### 1. auth.test.js ✅ (21 тестов)

#### POST /auth/login (10 тестов)
- ✅ Вернуть токены при корректных credentials
- ✅ Сохранить refresh токен в БД
- ✅ Вернуть 400 если отсутствует username
- ✅ Вернуть 400 если отсутствует password
- ✅ Вернуть 401 при неверном пароле
- ✅ Вернуть 401 при несуществующем пользователе
- ✅ Вернуть 403 для деактивированного пользователя
- ✅ Access токен должен иметь TTL 10 минут (600 сек)
- ✅ Refresh токен должен иметь TTL 14 дней (1209600 сек)
- ✅ Токены должны содержать корректные claims

#### POST /auth/refresh (6 тестов)
- ✅ Вернуть новый access токен при валидном refresh токене
- ✅ Новый access токен должен иметь актуальный iat
- ✅ Вернуть 400 если отсутствует refresh_token
- ✅ Вернуть 401 при невалидном refresh токене
- ✅ Вернуть 401 при отозванном refresh токене
- ✅ Вернуть 401 если использован access токен вместо refresh

#### POST /auth/revoke (5 тестов)
- ✅ Отозвать refresh токен
- ✅ Отозванный токен должен быть помечен в БД
- ✅ Отозванный токен нельзя использовать для refresh
- ✅ Вернуть 400 если отсутствует refresh_token
- ✅ Вернуть 400 при невалидном формате токена

- ✅ Вернуть 400 если отсутствует token

### 3. jwks.test.js ✅ (7 тестов)

#### GET /.well-known/jwks.json
- ✅ Вернуть JWKS с публичными ключами
- ✅ Ключ должен иметь корректную структуру JWK (kty, use, alg, kid, n, e)
- ✅ kid должен быть непустым
- ✅ Модуль (n) должен быть base64url строкой (без +/=)
- ✅ Экспонент (e) должен быть AQAB (стандарт RSA 65537)
- ✅ Возвращать один и тот же kid при повторных запросах (стабильность)
- ✅ Токены должны использовать kid из JWKS (согласованность)

## Покрытие функционала

### ✅ Аутентификация
- Login с username/password
- Выдача access и refresh токенов
- Валидация credentials
- Проверка активности пользователя
- Обработка ошибок (400, 401, 403)

### ✅ Управление токенами
- Refresh access токена через refresh токен
- Revoke (отзыв) refresh токена
- Проверка типа токена (access vs refresh)
- Сохранение refresh токенов в БД
- Маркировка отозванных токенов

### ✅ Introspection
- Валидация токенов
- Проверка подписи RS256
- Возврат claims (sub, username, type, exp, iat, iss, aud)
- Обработка невалидных и истекших токенов

### ✅ JWKS (Public Keys)
- Публикация публичных ключей в формате JWK
- Корректная структура (RSA, RS256)
- Стабильность kid между запросами
- Согласованность с токенами

### ✅ Health Check
- Проверка работоспособности сервиса
- Статус MongoDB подключения
- Доступность без аутентификации
- Быстрый ответ
